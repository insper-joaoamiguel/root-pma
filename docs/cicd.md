# CI/CD com GitHub Actions

O projeto utiliza GitHub Actions para automatizar o processo de integração contínua e entrega contínua da documentação.

## Pipeline implementada

A pipeline é executada automaticamente a cada push na branch principal.

Etapas:

1. Checkout do repositório
2. Instalação das dependências do MkDocs
3. Geração da documentação
4. Publicação automática no GitHub Pages

## Benefícios

- Deploy automatizado
- Redução de erros manuais
- Integração nativa com GitHub
- Versionamento da documentação
